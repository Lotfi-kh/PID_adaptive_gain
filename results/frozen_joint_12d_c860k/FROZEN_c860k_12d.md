# Deployed model, joint roll+pitch, 12-D, c860k

This is the model used for all the results in the thesis. It is the
"selected model" referred to in chapter 4.

## Model

| | |
|---|---|
| Checkpoint file | `td3_pid_860000_steps.zip` |
| SHA-256 | `c7d3a24b8aac2a5bb1fb56023d5af9571451962282f8d7d67999d073e40c94be` |
| Training steps | 860,000 |
| Observation dim | 12 |
| Action dim | 3 (shared, applied to roll and pitch) |
| Actor | Linear(12,64) -> ReLU -> Linear(64,64) -> ReLU -> Linear(64,3) -> Tanh |
| Parameters | 5187 |

### Observation layout (12 floats)

    [ roll, pitch,
      roll_rate, pitch_rate,
      roll_rate_err, pitch_rate_err,
      Kp_roll_n, Ki_roll_n, Kd_roll_n,
      Kp_pitch_n, Ki_pitch_n, Kd_pitch_n ]

The gains are normalized by the upper bound of each axis (KP/KI/KD_BOUNDS[1]).
Roll and pitch normalized gains are always equal because the action is
shared between the two axes.

### Reproducibility

The zip file is meant to be loaded with the 12-D version of
`envs/pybullet_pid_tuner_env.py` that is in this repository. An older
13-D environment (with the `step_prog` input) will not load it.

## Validation results

### Stable hover (`eval_stable_hover.py`, 300 hover samples)

Mean absolute action: **0.0407**. The policy stays almost still when the
drone is already stable, which is the wanted behaviour.

### Disturbance grid (`run_disturbance_grid.py`, roll + pitch)

The policy wins on 6 out of 6 conditions of the small grid, with 0 crashes.

- Peak |roll+pitch_rate| reduced by 53.8% on average.
- RMS roll+pitch_rate reduced by 8.5% on average.

The large 48-condition grid (the headline 42/48, -55.1% peak, -54.2%
recovery, 0/960 crashes) is generated by `run_aggressive_eval.py`.

### Sustained constant torque (`eval_sustained.py`)

Standing tilt on the disturbed axis (degrees), the policy vs the fixed
baseline:

| condition    | baseline | model | comment |
|---|---|---|---|
| low_roll     | 0.87     | 0.27  | better than baseline |
| low_pitch    | 0.87     | 0.32  | better than baseline |
| med_roll     | 3.42     | 0.97  | better than baseline |
| med_pitch    | 3.42     | 3.32  | comparable to baseline, see note below |
| combined_med | 3.42 / 3.42 | 1.03 / 1.03 | better than baseline |

Zero crashes in every condition. The integral gain stays alive (Ki between
0.035 and 0.068 across the conditions), and the proportional gain does not
run to its upper bound (Kp peak around 0.86, the bound is 1.72).

**Note on med_pitch.** On a sustained pitch torque the model is about the
same as the baseline, not better. It is not a regression and there are no
crashes, but the improvement we see on every other condition is not here.
This is the one limit of the model.
